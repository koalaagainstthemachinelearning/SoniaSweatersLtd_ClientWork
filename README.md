# Sonia Group – PO-to-ERP Order Entry Automation

A UiPath automation that reads customer Purchase Order (PO) PDFs, consolidates the order data into an audit spreadsheet, and then keys the orders into the **Logic ERP Solution** web application ("Order Entry by Matrix" screen) using UiPath's Modern (UIAutomationNext) activities.

> ⚠️ This README was generated by inspecting `Main.xaml` directly. Selectors, URLs, and file paths below are hardcoded in the workflow exactly as found — see [Known Limitations](#known-limitations--things-to-review) for what should be parameterized before reuse.

## What This Automation Does

1. Reads a merchandising configuration spreadsheet.
2. Prompts the operator for ERP login credentials.
3. Reads all PO PDF files from an input folder.
4. Parses a "PO_01" reference PDF to obtain size-ratio breakdown tables.
5. Parses one or more "PO_02"/"PO_2" PDFs to extract item, color, price, and quantity data, using the PO_01 ratios to split total quantity into size buckets (36, 38/40, 42/44, 46/48).
6. Writes all parsed rows to an **audit Excel workbook** for traceability before touching the ERP.
7. Logs into the Logic ERP Solution web system (Firefox) and, for every unique style/PO combination found, creates an order in the **Order Entry by Matrix** screen — filling in company, buyer, quantities, sizes, colors, dates, delivery country, and a reference product image.

## Process Flow

```
Read merchandise_input.xlsx (dt_Config)
        │
        ▼
Prompt for ERP username & password (Input Dialogs)
        │
        ▼
List all *.pdf in input folder
        │
        ├── Find file ending "PO_01"  → Read PDF text → split into per-page blocks (size-ratio tables)
        │
        └── Find file(s) ending "PO_02"/"PO_2"
                 │
                 ▼
        For each PO_02 file:
          Read PDF text
          Extract header (PO No, PO Date, ETD, Country)
          Extract item rows (Style Ref, Color, Unit Price, Total Qty)
          Look up size ratio in PO_01 blocks → compute Qty_36 / Qty_38-40 / Qty_42-44 / Qty_46-48
          Append row(s) to dt_AllJobs
        │
        ▼
Write dt_AllJobs to output\audit.xlsx (sheet "Audit")   ← audit checkpoint
        │
        ▼
Open Firefox → Logic ERP Solution → log in → Merchandising → Order Tracking
        │
        ▼
For each unique Style Ref in dt_AllJobs:
   Open "Order Entry by Matrix"
   Select company / buyer, enter style ref, pick matching style
   ⏸ Manual step: operator fills Team Leader, Dealing Merchant, SMVs, Yarn Quality
   Select production department, Save, upload reference image
   For each unique PO No under this style:
       Enter PO number & total quantity
       For each color: select color, set PO/ship dates, delivery country, quantity, confirm
       For each size bucket: register size name
       Inject computed quantities into the size/color matrix grid
       Save the order
```

## Prerequisites

- **UiPath Studio** (Windows) with a project compatible with the **UIAutomationNext** ("Modern" UI Automation) activity pack — the workflow uses `uix:NApplicationCard`, `uix:NClick`, `uix:NTypeInto`, `uix:NInjectJsScript`, `uix:NSelectItem`, `uix:NClosePopup`.
- **Mozilla Firefox** installed (the ERP interaction is scoped to `firefox.exe`; `BrowserType="Firefox"`).
- **Microsoft Excel** installed and licensed (activities use the classic `ui:ExcelApplicationScope`, which drives an Excel COM instance rather than the interop-free Excel activities).
- Network/VPN access to the ERP host: `182.160.125.188` (Logic ERP Solution, `erp_test` environment).
- Read access to the input folder and write access to the output folder described below.
- Valid ERP credentials for the target environment.

## Input/Output File Layout

The workflow currently uses **hardcoded absolute paths** (Windows, drive `E:`):

| Purpose | Path | Notes |
|---|---|---|
| Merchandising config workbook | `E:\Soniagroup\input\merchandise_input.xlsx` | Sheet `Sheet1`, read in full with headers |
| PO input PDFs | `E:\Soniagroup\input\*.pdf` | Must include exactly one "PO_01" file and one or more "PO_02"/"PO_2" files (see naming rule below) |
| Style reference images | `E:\Soniagroup\input\images\{StyleRef}.jpg` / `.jpeg` / `.png` | Filename must match the extracted Style Ref exactly |
| Audit output workbook | `E:\Soniagroup\output\audit.xlsx` | Sheet `Audit`; overwritten with the full `dt_AllJobs` table each run |

**PDF naming convention** (case-insensitive, matched against the filename without extension):
- Ends with `PO_01` → treated as the **ratio/size-breakdown reference document** (one file expected).
- Ends with `PO_02` or `PO_2` → treated as an **order PDF** to parse into rows (one or more files supported; the workflow loops over all matches).

## Configuration

`merchandise_input.xlsx` (`dt_Config`) is expected to supply, at minimum, the following columns (referenced by name inside the workflow's JS-injection steps, most of which are currently disabled — see below):

- `Team Leader`
- `Team Merchant`
- `Knit_SMV`, `Link_SMV`, `Trim_SMV`, `Fin_SMV`
- `Yarn Quality`
- `Prod. Dept`

> Only `Prod. Dept` (production department dropdown) is actively auto-filled in the current build. The others are wired up in `CommentOut` blocks (disabled) and are instead requested from the operator via a message box mid-run — see [Manual Steps](#manual-steps-during-a-run).

## Setup Instructions

1. **Clone/open** this project in UiPath Studio.
2. **Restore dependencies**: Project → Manage Packages → ensure the packages listed above are installed at matching or newer versions.
3. **Recreate or verify folder paths.** Either:
   - Create the exact folder structure the workflow expects (`E:\Soniagroup\input`, `E:\Soniagroup\input\images`, `E:\Soniagroup\output`), **or**
   - Edit the `folderPath` variable default and the two `ExcelApplicationScope.WorkbookPath` values (input config workbook and output audit workbook) to match your environment.
4. **Verify ERP reachability**: confirm you can reach `http://182.160.125.188/erp_test/login.php` from the machine that will run the robot, and that you have valid credentials.
5. **Confirm UI selectors still match** the ERP's current DOM/version — the automation relies on fixed element IDs (e.g. `txt_userid`, `txt_password`, `cbo_company_name`, `cbo_buyer_name`, `txt_style_ref`, `save1`, `save2`, `txt_colorSizeQty_{n}_{n}`, etc.) inside an `IFRAME#page_container`. Any ERP update to these IDs will break the automation.
6. **Populate the input folder** with:
   - `merchandise_input.xlsx`
   - One "PO_01" PDF and one or more "PO_02"/"PO_2" PDFs
   - A matching product image per unique Style Ref in `input\images\`
7. **Run in Debug** first against a test/staging ERP environment before pointing at production.

## Running the Automation

1. Open the project in UiPath Studio (or run via UiPath Assistant/Orchestrator if published).
2. Execute `Main.xaml`.
3. When prompted, enter:
   - ERP username
   - ERP password
4. The robot will:
   - Build the audit spreadsheet automatically (no interaction needed).
   - Launch Firefox and log into the ERP.
   - Pause at a message box (see below) once per Style Ref for manual field entry.
5. Monitor the run — UI automation against a live ERP is timing-sensitive; keep the automated browser window focused and avoid touching the keyboard/mouse while it runs (several steps use `SendHotkey`/clipboard paste, which are sensitive to focus changes).

## Manual Steps During a Run

For **each unique Style Ref**, after the style is selected, the robot displays a message box and pauses for the operator to manually enter, inside the ERP UI:

1. Team Leader
2. Dealing Merchant
3. Knit SMV
4. Link SMV
5. Trim SMV
6. Fin SMV
7. Yarn Quality

Click **OK** in the message box once these fields have been filled in the ERP to let the robot continue (it will then set the production department automatically and proceed to save the order, upload the image, and enter PO/quantity/color/size data).

> These fields have working (but disabled) `InjectJsScript` automation blocks already present in the XAML, wired to `dt_Config` columns `Team Leader`, `Team Merchant`, `Knit_SMV`, `Link_SMV`, `Trim_SMV`, and `Yarn Quality`. They can be re-enabled by removing the surrounding `CommentOut` activities once the corresponding ERP element IDs (`cbo_team_leader`, `cbo_dealing_merchant`, `smv_1`, `cutsmv_1`, `trimsmv_1`, `finsmv_1`, `txt_yarn_quality`) are confirmed stable.

## Data Extracted per PO

For each order line, the automation extracts/derives:

| Field | Source |
|---|---|
| `PO_No` | Regex on PO text: `Order n°/no : (\d+)` |
| `PO_Date` | Regex: `Date DD/MM/YYYY` |
| `ETD` | Regex: `ETD : Expected DD/MM/YYYY` |
| `Delivery_Country` | Regex on address block, title-cased |
| `Style_Ref` | First captured item code per PDF (e.g. matches pattern like `XX-XXX1234`) |
| `Color` | Parsed alongside style/garment description line |
| `Unit_Price` | Regex: `$US <price> / Unité <qty>` |
| `Total_Qty` | Same match as unit price |
| `Qty_36`, `Qty_38_40`, `Qty_42_44`, `Qty_46_48` | Proportionally split from `Total_Qty` using the ratio found in the matching PO_01 block for that style/color |

All rows are accumulated into a single `dt_AllJobs` DataTable, which is both the audit export and the driver for ERP data entry.

## Known Limitations / Things to Review

- **Hardcoded absolute paths** (`E:\Soniagroup\...`) — not portable across machines; should be moved to `config.xlsx`, arguments, or Orchestrator assets.
- **Hardcoded ERP URL/IP** (`182.160.125.188`) baked into every UI target's `BrowserURL`/`ScopeSelectorArgument` — will need updating if the environment changes (test → production, IP → hostname, etc.).
- **Hardcoded business values** in the automation itself, not the config sheet: buyer `"Systeme U [Cobalt]"`, company `"Sonia and Sweaters Ltd."`, size list `{"36","38/40","42/44","46/48"}`, PO item dropdown option `"1/4 ZipTroyer neck sweater"`. These will need to become parameters/config rows for use with other buyers, companies, or garment types.
- **Regex-based PDF parsing** is tightly coupled to one PO template's wording (French labels like `Order n°`, `Unité`, `Colori`). A differently formatted PO PDF will silently fail to match and throw (`"No items found in..."` / `"Item/price mismatch..."`).
- **Plaintext credential handling** — username/password are held only in workflow variables in memory (not written to disk within this workflow), but are supplied via a non-masked `InputDialog` for username and a masked one for password; ensure this is run in a controlled/unattended context appropriate for credential handling in your organization (see [Security Notes](#security-notes)).
- **Several JS-injection steps for SMV/Team fields are disabled** (`CommentOut`) in favor of a manual message-box step — this is a partially-automated workaround, not a finished flow.
- **Fixed `Delay` activities** (1–5 seconds) are used throughout instead of explicit wait-for-element conditions, which can make the bot fragile under slow network/ERP conditions.
- **No retry/error-recovery logic** around the ERP save/upload steps — a failed save mid-loop will stop the run without resuming from the last successful Style/PO combination.
- **Single monolithic `Main.xaml`** — recommend refactoring into invoked sub-workflows (e.g. `ExtractPOData.xaml`, `WriteAuditFile.xaml`, `EnterOrderInERP.xaml`) for readability, reusability, and easier testing.

## Troubleshooting

| Symptom | Likely Cause |
|---|---|
| `"No items found in: <file>"` | The PO_02/PO_2 PDF doesn't match the expected item/description regex — check PDF text layout or template changes. |
| `"Item/price mismatch: X items vs Y prices"` | Item count and price count extracted from the PDF don't line up — usually a formatting change in the source PDF. |
| `WARNING: Ratio not found for <style> | <color>` (console output) | No matching size-ratio block was found in the PO_01 file for that style/color combination; quantities will default to 0 for all size buckets. |
| Robot can't find ERP fields / selectors fail | ERP UI has changed, or the automated Firefox window lost focus/was resized; verify `ScopeSelectorArgument`/`FullSelectorArgument` targets still exist in the current ERP version. |
| Image not attached | No file named exactly `{StyleRef}.jpg`/`.jpeg`/`.png` exists in `input\images\`. |
| Login fails | Confirm ERP host `182.160.125.188` is reachable and credentials entered in the Input Dialogs are correct. |

## Security Notes

- Credentials are collected at runtime via UiPath `InputDialog` activities rather than being stored in the project — avoid hardcoding credentials in `Main.xaml` or in `merchandise_input.xlsx`.
- If deploying to Orchestrator/unattended robots, consider replacing the `InputDialog` prompts with **Orchestrator Assets/Credential Assets** so no one has to type credentials interactively.
- The ERP endpoint is accessed over plain HTTP (`182.160.125.188`, no HTTPS) — treat this as an internal/trusted-network-only integration, and avoid running it over an untrusted network without a VPN or equivalent protection.
